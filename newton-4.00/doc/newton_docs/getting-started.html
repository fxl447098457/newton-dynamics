1. Requirements

Newton Dynamics 4.00 requires:

  • C++11 compiler - MSVC 2017+, GCC 7+, Clang 5+, or Apple Clang
  • CMake 3.10+ - for build configuration
  • Platform - Windows, Linux, macOS, or ARM (Android, iOS, Raspberry Pi)

Optional Dependencies

  • OpenGL + GLFW - for the demo application (ndSandbox)
  • OpenCL SDK - for GPU acceleration (Windows only)

2. Building Newton

Windows (Visual Studio)

# Create build directory
mkdir build
cd build

# Configure with CMake
cmake .. -G "Visual Studio 17 2022" -A x64

# Build Release configuration
cmake --build . --config Release

Linux / macOS

# Create build directory
mkdir build && cd build

# Configure with CMake
cmake .. -DCMAKE_BUILD_TYPE=Release

# Build
cmake --build . -j$(nproc)

Build Output

After building, you'll find:

  • ndNewton.dll / libndNewton.so - the main physics library
  • ndSandbox - demo application (if enabled)
  • Header files in sdk/ subdirectories

3. CMake Options

Configure Newton with these CMake options:

Option Default Description
NEWTON_BUILD_SANDBOX_DEMOS ON Build the demo applications
NEWTON_BUILD_SHARED_LIBS ON Build as shared library (.dll/.so)
NEWTON_DOUBLE_PRECISION OFF Use double precision floats
NEWTON_ENABLE_AVX2 ON Enable AVX2 SIMD (x86 only)
NEWTON_BUILD_SINGLE_THREADED OFF Disable multithreading
NEWTON_BUILD_PROFILER OFF Enable Tracy profiler support
NEWTON_STATIC_RUNTIME_LIBRARIES OFF Use static runtime (Windows /MT)

Example: Static Library with Double Precision

cmake .. -DNEWTON_BUILD_SHARED_LIBS=OFF -DNEWTON_DOUBLE_PRECISION=ON

4. SDK Structure

The Newton SDK is organized into layered modules:

newton-4.00/
├── sdk/
│   ├── dCore/          # Math, containers, threading (foundation)
│   ├── dCollision/     # Bodies, shapes, collision detection
│   │   └── dMesh/      # Mesh processing utilities
│   └── dNewton/        # Physics world, solver, dynamics
│       ├── dJoints/    # All joint/constraint types
│       ├── dIkSolver/  # Inverse kinematics
│       └── dModels/
│           └── dVehicle/  # Vehicle simulation
├── applications/
│   └── ndSandbox/      # Demo application
└── tests/              # Unit tests (GoogleTest)

Key Header Files

Header Purpose
ndNewton.h Main include - brings in entire physics API
ndWorld.h Physics world and simulation control
ndBody.h Base body class and body types
ndShape*.h Collision shape definitions
ndJoint*.h Joint/constraint definitions

5. Linking to Your Project

CMake (Recommended)

# Add Newton as a subdirectory
add_subdirectory(newton-4.00)

# Link to your target
target_link_libraries(MyGame PRIVATE ndNewton)

# Include directories are set automatically

Manual Linking

Add include paths:

newton-4.00/sdk/dCore
newton-4.00/sdk/dCollision
newton-4.00/sdk/dNewton
newton-4.00/sdk/dNewton/dJoints
newton-4.00/sdk/dNewton/dModels
newton-4.00/sdk/dNewton/dIkSolver

Link against ndNewton library.

6. Hello World Example

A minimal Newton application that creates a world, adds a falling sphere, and simulates one second:

#include <ndNewton.h>
#include <cstdio>

int main()
{
    // Create physics world
    ndWorld world;
    world.SetSubSteps(2);

    // Create a dynamic body
    ndBodyDynamic* body = new ndBodyDynamic();

    // Set position 10 meters up
    ndMatrix matrix(ndGetIdentityMatrix());
    matrix.m_posit = ndVector(0.0f, 10.0f, 0.0f, 1.0f);
    body->SetMatrix(matrix);

    // Attach sphere collision (radius = 1 meter)
    ndShapeInstance sphere(new ndShapeSphere(1.0f));
    body->SetCollisionShape(sphere);

    // Set mass (10 kg) - auto-computes inertia from shape
    body->SetMassMatrix(10.0f, sphere);

    // Add gravity callback
    body->SetNotifyCallback(new ndBodyNotify(ndVector(0.0f, -9.8f, 0.0f, 0.0f)));

    // Add body to world (using shared pointer)
    ndSharedPtr<ndBody> bodyPtr(body);
    world.AddBody(bodyPtr);

    // Simulate 1 second at 60 Hz
    for (int i = 0; i < 60; i++)
    {
        world.Update(1.0f / 60.0f);
        world.Sync();

        ndVector pos = body->GetMatrix().m_posit;
        printf("Frame %d: Y = %.2f m\n", i, pos.m_y);
    }

    return 0;
}
Tip: Newton uses shared pointers (ndSharedPtr) for automatic memory management. Bodies, joints, and models are reference-counted and automatically deleted when no longer referenced.

7. The Simulation Loop

A typical Newton simulation loop:

void GameLoop(ndWorld& world, float deltaTime)
{
    // 1. Apply forces/inputs to bodies
    ApplyPlayerInput();

    // 2. Step the physics simulation
    world.Update(deltaTime);

    // 3. Wait for physics thread to complete
    world.Sync();

    // 4. Read body transforms for rendering
    UpdateGraphics();
}

Key Methods

Method Description
Update(timestep) Advances simulation by timestep seconds. Runs asynchronously.
Sync() Blocks until physics update completes. Call before reading transforms.
SetSubSteps(n) Divides timestep into n sub-steps for stability. Default: 1.
SetThreadCount(n) Sets number of worker threads. Default: auto-detect.
Note: Use a fixed timestep (e.g., 1/60) for deterministic simulation. Variable timesteps can cause instability and non-reproducible results.
